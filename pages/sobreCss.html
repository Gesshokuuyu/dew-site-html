Tecnologia #2

{ CSS }

Cascading Style Sheets — a linguagem que transforma estrutura em design, cor e movimento.

O que é CSS?

CSS é a linguagem usada para estilizar documentos HTML. Com ela você controla cores, fontes, espaçamentos, layouts e até animações — tudo que torna uma página visualmente agradável.

O nome Cascading (cascata) vem do modo como regras são aplicadas: estilos se sobrepõem em uma ordem de prioridade, permitindo herança e especificidade.

Você sabia? CSS foi proposto por Håkon Wium Lie em 1994. O CSS3 trouxe módulos independentes como Flexbox, Grid, animações e variáveis nativas.

Anatomia de uma Regra CSS

Toda regra CSS é composta por seletor, propriedade e valor:

h1 {
  color: #ff6b35;
  font-size: 2rem;
}
  • Seletor (qual elemento estilizar)
  • Propriedade (o que mudar)
  • Valor Estilizado(tamanho, cor, etc.)

Tipos de Seletores

Os seletores determinam quais elementos recebem um estilo:

elemento

Seleciona pela tag HTML. Ex: p { color: red } estiliza todos os parágrafos.

.classe

Seleciona pelo atributo class. Pode ser reutilizado em múltiplos elementos.

#id

Seleciona pelo atributo id. Deve ser único por página, alta especificidade.

pai > filho

Seleciona elementos filhos diretos. Permite estilizar com base na hierarquia do HTML.

:hover

Pseudo-classe que dispara quando o mouse passa sobre o elemento — ideal para interações.

@media

At-rule para responsividade. Aplica estilos diferentes conforme o tamanho da tela.

Box Model

Todo elemento HTML é tratado como uma caixa com quatro camadas:

margin
border
padding
conteúdo
  • Content — o conteúdo real (texto, imagem).
  • Padding — espaço interno entre o conteúdo e a borda.
  • Border — a borda ao redor do elemento.
  • Margin — espaço externo que separa o elemento dos vizinhos.

Flexbox

O Flexbox organiza elementos em uma linha ou coluna com total controle de alinhamento:

display: flex

Ativa o contexto Flexbox no elemento pai. Os filhos diretos tornam-se flex items.

flex-direction

Define o eixo principal: row (horizontal) ou column (vertical).

justify-content

Alinha os itens no eixo principal. Ex: center, space-between.

align-items

Alinha os itens no eixo cruzado (perpendicular). Ex: center, stretch.

flex-wrap

Permite que os itens quebrem para a próxima linha quando não há espaço suficiente.

gap

Define o espaçamento entre os itens flex sem precisar usar margin.

Grid Layout

O Grid cria layouts bidimensionais — linhas e colunas ao mesmo tempo:

display: grid

Ativa o contexto Grid no elemento pai. Os filhos são posicionados nas células da grade.

grid-template-columns

Define o número e tamanho das colunas. Ex: repeat(3, 1fr) cria 3 colunas iguais.

grid-template-rows

Define o número e altura das linhas. Pode usar valores fixos, fr ou auto.

grid-column / grid-row

Faz um item ocupar múltiplas colunas ou linhas. Ex: grid-column: 1 / 3.

gap

Define o espaço entre linhas e colunas da grade de forma simplificada.

place-items

Atalho para align-items e justify-items ao mesmo tempo.

Animações

CSS permite criar movimentos e transições sem nenhuma linha de JavaScript:

transition

Suaviza a mudança entre dois estados. Ex: transition: color 0.3s ease em um hover.

@keyframes

Define os fotogramas de uma animação, descrevendo o estado inicial, intermediário e final.

animation

Aplica um @keyframes ao elemento. Controla duração, repetição e timing.

transform

Modifica forma ou posição visualmente. Ex: scale(), rotate(), translateX().

animation-delay

Aguarda um tempo antes de iniciar a animação — útil para efeitos em cascata.

animation-fill-mode

forwards mantém o estado final da animação após ela terminar.

Medidas: Fixas vs Relativas

Escolher a unidade correta afeta responsividade, acessibilidade e manutenção do código:

px (pixel)

Fixo. Não escala com o zoom ou preferências do usuário. Útil para bordas e detalhes precisos, mas evitar em fontes.

rem (root em)

Relativo. Baseado no tamanho da fonte do elemento <html>. Melhor opção para layouts e tipografia responsivos.

em (element em)

Relativo. Baseado no tamanho da fonte do elemento pai. Útil para componentes reutilizáveis com espaçamento proporcional.

% (percentual)

Relativo. Baseado no elemento pai. Perfeito para larguras fluidas em layouts responsivos.

vh / vw (viewport)

Relativo. Baseado no tamanho da janela (altura/largura). Ótimo para seções fullscreen e heróis.

clamp()

Fluido. Define mín, preferido e máx. Ex: clamp(1rem, 5vw, 3rem) — tipografia responsiva sem @media.

Mobile First

Mobile First é uma abordagem em que você escreve os estilos base para telas pequenas e usa @media para adicionar complexidade em telas maiores.

Com min-width como ponto de partida, o CSS cresce progressivamente, resultando em código mais limpo, performático e acessível em qualquer dispositivo.

Dica: Comece sempre pelo estilo mobile, sem @media. Só envolva em breakpoints o que precisa mudar em telas maiores — nunca o contrário.
